# WebDriver + Java + TestNG

1. Select a system under test (preferable: web application being tested during your project activities);

2. Agree with your mentor and document 3 scenarios to automate. The scenarios should be linear (no need to implement complex logic for now) and contain 8-10 simple steps;

3. Use WebDriver API as much as possible (browser navigation, clicks, switchTo and etc.);

4. Use several locator strategies i.e. different types of locators (and select the most suitable in your case);

5. Experiment with waits (implicit and explicit);

6. Extend your scenario with usage of Page Object / Page Factory patterns.

 If option #1 can’t be implemented by you due to any technical/project security reasons (project NDA) – use tasks from the eLearning module assigned to the learning path.
